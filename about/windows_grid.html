Hi!

I'm Dimitar Sotirov, a student and a game programmer at Gameloft, Sofia.

In programming I value:
* High-performance - no need to accelerate climate change by wasting CPU clock cycles.
* Quality - code with lots of comments. Maximizing readibility, minimizing dependencies and lines of code, because every LOC is a liability.
* Discipline. As in any engineering field, the final software product should be robust and maintainable.

I'm not saying this way of programming is THE way to program... :)
It's just suited better in the context of stuff I'm interested in - libraries used for embedded software, game engines or in general - code that other developers use.

That's why "it just works" is not enough for me (as it may be for application developers) and I need to make sure what I write stands the test of time. For this reason, I tend to avoid modern programming practices, which are aimed at fast iteration, but tend to have poor performance, raise complexity and code coupling significantly - such as OOP, modern C++, etc.
